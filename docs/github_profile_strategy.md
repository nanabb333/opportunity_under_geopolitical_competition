# GitHub Profile Strategy

## Purpose

This document recommends how to present the three-repository portfolio on GitHub so recruiters can understand the progression quickly.

## Pinned Repositories

Recommended pinned order:

1. `opportunity_under_geopolitical_competition`
2. `Taiwan_Geopolitical_Risk_Project`
3. `AI_Geopolitical_Risk_Dashboard`

If GitHub allows additional pinned repositories, add one supporting technical project only if it reinforces data engineering, analytics automation, or product delivery.

## Repository Ordering

The profile should lead with the strongest narrative sequence:

```text
Research Foundation
        |
        v
Analytics Engine
        |
        v
Decision Support Product
```

Avoid presenting the dashboard first if the recruiter would miss the research and analytics depth behind it.

## Profile README Structure

Recommended GitHub profile README sections:

1. One-sentence positioning
2. Portfolio roadmap
3. Pinned project overview
4. Skills demonstrated
5. Current focus
6. Contact or links

Suggested positioning:

> I build geopolitical risk analytics projects that turn research, structured data, and scenario evidence into decision-support tools.

## Recruiter Optimisation

A recruiter should be able to understand within one minute:

- the domain: geopolitical risk and strategic technology
- the method: research-backed analytics and scenario evidence
- the product direction: decision-support dashboards and assistants
- the role fit: business analytics, data analytics, analytics engineering, AI product analytics, or geopolitical risk analysis

## Avoid

Avoid claims that the portfolio predicts markets, provides investment advice, or forecasts geopolitical outcomes. The strongest positioning is disciplined evidence synthesis and decision-support analytics.
