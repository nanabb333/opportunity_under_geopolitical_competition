# Roadmap V2

## Phase 1: Research Foundation

### Goals

- Preserve dissertation framing.
- Maintain transparent research design.
- Establish event-study evidence and limitations.

### Deliverables

- dissertation research framing
- event-study dataset and outputs
- theory, research design, and protocol freeze documents
- dissertation figures and tables

### Limitations

- Small validated sample.
- Mixed empirical results.
- Descriptive interpretation only.
- No forecast or investment advice.

## Phase 2: Scenario Intelligence Product

### Goals

- Convert the dissertation evidence base into a deterministic scenario-analysis prototype.
- Support historical analogue comparison and observed pathway summaries.
- Communicate outputs through a static dashboard.

### Deliverables

- historical analogue dataset
- similarity engine
- scenario query demo
- observed pathway engine
- interactive scenario assistant
- dashboard evidence view
- portfolio and recruiter documentation

### Limitations

- Scenario inputs are predefined.
- Dataset coverage remains limited.
- Similarity scoring is qualitative and deterministic.
- Outputs are historical comparisons, not predictions.

## Phase 3: Intelligence System

### Goals

- Define the operating model for continuous event monitoring.
- Separate source monitoring, candidate review, approved dataset integration, and analytics refresh.
- Improve scalability without weakening human review.

### Deliverables

- intelligence system architecture
- source registry
- source registry methodology
- event monitoring framework
- dataset update protocol
- analyst workbench guide

### Limitations

- No automated scraping yet.
- No live feeds.
- Human review remains required.
- The system architecture is a foundation, not a fully automated intelligence platform.

## Phase 4: Automation Expansion

### Goals

- Automate parts of candidate event collection and review support.
- Improve repeatability of monitoring and update workflows.
- Preserve separation between candidate events and approved dataset rows.

### Deliverables

- monitored source configuration
- scheduled candidate generation
- duplicate detection
- candidate queue summaries
- stronger validation reports
- automated coverage refresh

### Limitations

- Automation should not approve events without review.
- Source access and licensing constraints must be respected.
- Automated collection may miss context or introduce noise.

## Phase 5: LLM-Augmented Assistant (Future)

### Goals

- Add language assistance for summarising reviewed evidence and helping analysts query the evidence base.
- Use LLMs only after the deterministic data model, validation workflow, and human-review process are stable.

### Deliverables

- evidence-grounded retrieval assistant
- citation-aware source summaries
- analyst drafting support
- scenario explanation layer

### Limitations

- LLM outputs must not be treated as verified facts without source review.
- No investment recommendations.
- No forecast accuracy claims.
- Model use must preserve data provenance, citations, and human approval checkpoints.
