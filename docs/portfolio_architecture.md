# Portfolio Architecture

## Purpose

This portfolio is organised as a progression from academic research to analytics engineering to decision-support product design.

```text
Repo 1: opportunity_under_geopolitical_competition
Research Foundation
        |
        v
Repo 2: Taiwan_Geopolitical_Risk_Project
Analytics Engine
        |
        v
Repo 3: AI_Geopolitical_Risk_Dashboard
Decision Support Product
```

## Repo 1: Research Foundation

`opportunity_under_geopolitical_competition` establishes the research base. It shows how a dissertation question can become a structured analytical system.

Core role:

- define the research question
- preserve dissertation theory and event-study framing
- build a historical analogue dataset
- create reproducible validation and analysis scripts
- document limitations and research boundaries
- demonstrate evidence synthesis without forecasting or investment advice

Portfolio signal:

This repo shows research discipline, careful coding, reproducibility, and the ability to translate academic work into a structured analytical foundation.

## Repo 2: Analytics Engine

`Taiwan_Geopolitical_Risk_Project` should function as the analytics engine layer. It builds from the research foundation by applying geopolitical risk logic to a more focused Taiwan-centred analytical workflow.

Core role:

- operationalise regional geopolitical risk indicators
- transform structured event logic into reusable analytics
- emphasize data processing, scoring, comparison, and interpretation
- bridge research design and product-oriented outputs

Portfolio signal:

This repo should show that the research foundation can be turned into an analytical engine: structured inputs, repeatable transformations, and interpretable outputs.

## Repo 3: Decision Support Product

`AI_Geopolitical_Risk_Dashboard` should function as the product layer. It packages the analytical logic into an interface that a nontechnical reviewer can understand quickly.

Core role:

- present scenario evidence through a dashboard or assistant-style interface
- make outputs usable for decision support
- show product thinking, UX judgment, and communication design
- demonstrate how AI-assisted or AI-adjacent workflows can be bounded by evidence and limitations

Portfolio signal:

This repo should show productization: the ability to turn research and analytics into a usable, explainable decision-support experience.

## How the Repositories Build on Each Other

The intended progression is:

1. Repo 1 asks and structures the research problem.
2. Repo 2 turns the research logic into a focused analytics engine.
3. Repo 3 packages the analytics into a product-facing decision-support interface.

Together, the portfolio demonstrates a full value chain:

```text
Research question
        |
        v
Structured evidence base
        |
        v
Deterministic analytics
        |
        v
Scenario intelligence
        |
        v
Decision-support interface
```

## Portfolio Boundary

The portfolio should be presented as evidence synthesis and decision-support analytics. It should not claim predictive accuracy, expected returns, investment advice, or automated geopolitical forecasting.
