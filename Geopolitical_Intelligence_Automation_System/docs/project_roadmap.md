# Project Roadmap

## Phase 1: System Design

Goal:

Define the intelligence automation architecture and operating model.

Deliverables:

- system architecture
- workflow architecture
- intelligence lifecycle
- project roadmap

Limitations:

- no live automation
- no dashboard implementation
- no LLM calls
- no forecasting

## Phase 2: Data Architecture

Goal:

Create data schemas for the core workflow.

Deliverables:

- `data/source_registry.csv`
- `data/candidate_events.csv`
- `data/approved_events.csv`
- `data/analyst_briefs.csv`

Limitations:

- schemas only
- no approved intelligence records yet
- no automated data collection

## Phase 3: Portfolio Positioning

Goal:

Position this repository as the fourth project in a broader geopolitical risk analytics portfolio.

Deliverables:

- portfolio positioning document
- clear relationship to research, analytics engineering, and decision-support repositories

Limitations:

- positioning does not imply production readiness
- claims remain portfolio-focused and evidence-based

## Phase 4: Automation Prototype

Goal:

Build local scripts that validate schemas, inspect queue status, and generate operational summaries.

Potential deliverables:

- schema validator
- queue status report
- approved-record validation
- generated operations report

Limitations:

- no scraping
- no automated approval
- no forecasting

## Phase 5: Dashboard Prototype

Goal:

Create a static dashboard that displays workflow status, queue counts, approved record counts, analyst briefs, and limitations.

Potential deliverables:

- static HTML dashboard
- local JSON output
- operational status cards
- evidence traceability view

Limitations:

- no backend required
- no investment advice
- no predictive scoring

## Phase 6: AI Workflow Expansion

Goal:

Explore AI-assisted workflow design after the deterministic pipeline is stable.

Potential deliverables:

- prompt design documentation
- human-in-the-loop review design
- evidence-grounded summarisation prototype

Limitations:

- no unsupervised approval
- no unsupported claims
- no LLM output treated as verified evidence
