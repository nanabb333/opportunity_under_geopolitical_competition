# Portfolio Positioning

## Purpose

This repository is designed as the fourth project in a geopolitical risk analytics portfolio. It complements the earlier repositories by focusing on automation architecture and intelligence operations.

## Portfolio Sequence

```text
Repo 1: Research
        |
        v
Repo 2: Analytics Engineering
        |
        v
Repo 3: Decision-Support Product
        |
        v
Repo 4: Automation System
```

## Repo 1: Research

Role:

Establishes the research foundation. It defines the core geopolitical risk question, preserves academic framing, builds a historical analogue evidence base, and documents limitations.

What it demonstrates:

- research design
- event-study thinking
- evidence discipline
- historical analogue reasoning
- transparent limitations

## Repo 2: Analytics Engineering

Role:

Turns research logic into reusable analytical workflows. It should show structured processing, repeatable calculations, and interpretable data outputs.

What it demonstrates:

- data pipeline design
- feature construction
- scenario comparison
- validation
- reproducible analytics

## Repo 3: Decision-Support Product

Role:

Packages analytics into a user-facing decision-support experience. It should show how complex geopolitical evidence can be communicated clearly.

What it demonstrates:

- dashboard design
- stakeholder communication
- product thinking
- evidence display
- responsible AI/product boundaries

## Repo 4: Automation System

Role:

Defines the operating system for continuously processing geopolitical information through source monitoring, candidate capture, review, approval, analogue search, evidence synthesis, analyst briefs, and dashboard publication.

What it demonstrates:

- AI workflow design
- automation architecture
- intelligence operations
- dataset governance
- human-in-the-loop review
- decision-support workflow design

## Portfolio Narrative

The portfolio progression is:

```text
Research credibility
        |
        v
Analytics implementation
        |
        v
Product communication
        |
        v
Operational automation
```

This repository completes the narrative by showing how the earlier research and product layers could be managed as a repeatable intelligence workflow.

## Positioning Boundary

This repository should be described as an automation and intelligence operations foundation. It should not be described as a forecasting system, trading system, or investment recommendation platform.
